Brand Guidelines

The complete visual identity and technical implementation standards for the digital platform division of Annex Brands. Every color, typeface, spacing value, animation curve, and component pattern — defined as executable specifications.

01 Overview

NLS Digital powers the technology layer beneath Annex Brands' seven franchise brands and 850+ retail locations — from instant quoting engines and shipment tracking to API integrations and white-label solutions. These guidelines exist as executable specifications. The brand is the code.

2.5 Dark / Light Mode

Dark mode is the default expression. Light mode uses semantic aliases that remap to lighter values. The system detects prefers-color-scheme automatically but can be overridden with data-theme="light" on the root element.

/* Semantic aliases — remap per theme */
[data-theme="light"] {
  --bg-primary:     var(--nls-light-bg);       /* #F4F6F8 */
  --bg-surface:     var(--nls-light-surface);  /* #FFFFFF */
  --border-default: var(--nls-light-border);   /* #DDE3E8 */
  --text-primary:   var(--nls-light-text);     /* #1A2530 */
  --text-secondary: var(--nls-light-muted);    /* #5A6872 */
}

@media (prefers-color-scheme: light) {
  :root:not([data-theme]) { /* same overrides */ }
}

03 Typography

A three-tier system: a display face for impact, a body face for readability, and a monospace face for technical content.

3.1 Font Stack

RoleFamilyFallback ChainRationale
Display / Headings Plus Jakarta Sans Futura → Trebuchet MS → sans-serif Geometric sans with warmth. Futura fallback connects to Annex heritage.
Body / UI Inter Helvetica Neue → Arial → sans-serif Purpose-built for screens. Tall x-height, unambiguous at 12–16px.
Code / Data JetBrains Mono Fira Code → SF Mono → Consolas → monospace Distinct 1/l/I and 0/O glyphs. Programming ligatures. API-ready.

3.5 Font Loading Strategy

Fonts are loaded via Google Fonts with font-display: swap to prevent FOIT (flash of invisible text). Critical weights are preloaded. A .fonts-loading class on <html> provides a system-font fallback until fonts resolve, preventing layout shift.

<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<script>
  if ('fonts' in document) {
    Promise.all([
      document.fonts.load('700 1em "Plus Jakarta Sans"'),
      document.fonts.load('400 1em "Inter"'),
    ]).then(() => {
      document.documentElement.classList.remove('fonts-loading');
      document.documentElement.classList.add('fonts-loaded');
    });
  }
</script>

8.4 Staggered Entry

Apply .nls-stagger to a parent to auto-delay children's entrance animation by 50ms increments (capped at 400ms for 9+ children).

.nls-stagger > * {
  animation: nls-fade-in-up 300ms var(--ease-enter) both;
}
.nls-stagger > *:nth-child(1) { animation-delay: 0ms;   }
.nls-stagger > *:nth-child(2) { animation-delay: 50ms;  }
.nls-stagger > *:nth-child(3) { animation-delay: 100ms; }
/* ... through nth-child(8) at 350ms */

8.5 Scroll-Triggered Reveal

// Add data-nls-reveal to any element for scroll-triggered entrance
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.dataset.nlsReveal = 'visible';
        observer.unobserve(entry.target);
      }
    });
  },
  { threshold: 0.15, rootMargin: '0px 0px -60px 0px' }
);
document.querySelectorAll('[data-nls-reveal]').forEach(el => observer.observe(el));

12.2 Change Process

All modifications require review by the NLS Digital Design Lead and Engineering Lead. Token changes must propagate through the design token pipeline (JSON → CSS / SCSS / JS) before deployment. Semantic versioning applies: major = breaking visual changes, minor = new components/tokens, patch = corrections.

The single source of truth is nls-tokens.json. Platform-specific formats (CSS custom properties, SCSS variables, iOS/Android constants, Figma variables) are generated via Style Dictionary. Direct editing of derived formats is prohibited.
